# Step by Step Delivery Template

1. Confirm module scope and exclusions.
2. Finalize module requirements.
3. Finalize module design decisions.
4. Implement mapped scope. Any database schema/index/data-migration change in this step must be added as an executable PostgreSQL script under ad-docs/database-script before marking implementation complete.
5. Execute module acceptance tests.
6. Capture release-readiness evidence.
7. Update risk, decision, and open-question logs.
